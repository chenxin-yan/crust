Use `renderManPage()` to produce a **troff `-man`** document from your command tree (the same internal `CommandNode` shape used for `--help`). Use `manPagePlugin()` to expose a subcommand (default `man`) that prints that document to stdout or writes it with `--output`.

## Usage

### Programmatic

```ts
import { writeFileSync } from "node:fs";
import { renderManPage } from "@crustjs/plugins";
import { app } from "./cli.js";

const troff = renderManPage(app._node, {
  date: "April 2026",
  source: "my-cli",
  manual: "User Commands",
  extraSectionsAfter: [
    { title: "SEE ALSO", body: "https://example.com/docs" },
  ],
});

writeFileSync("man/my-cli.1", troff, "utf8");
```

<Callout type="warn">
  `CommandNode` is the internal shape built by `Crust`. Expose your root node from your CLI module (for example export the builder’s `._node` in application code) or call `renderManPage` from code that already holds the root during setup.
</Callout>

### Runtime subcommand

```ts
import { Crust } from "@crustjs/core";
import { manPagePlugin } from "@crustjs/plugins";

const main = new Crust("my-cli")
  .meta({ description: "My CLI" })
  .use(manPagePlugin({ command: "man" }))
  .run(() => { /* ... */ });

await main.execute();
```

```sh
my-cli man > man/my-cli.1
my-cli man --output man/my-cli.1
```

Preview with `groff -man -Tutf8 -P-c man/my-cli.1 | less` (or your platform’s `man` for local files).

## Options

`ManPageGeneratorOptions` (passed to `renderManPage` and merged into `manPagePlugin`):

| Field | Purpose |
| ----- | ------- |
| `section` | Manual section (default `1`) |
| `title`, `date`, `source`, `manual` | `.TH` line fields |
| `extraSectionsBefore` / `extraSectionsAfter` | `{ title, body }[]` plain text, escaped for troff |
| `rawTroffSectionsBefore` / `rawTroffSectionsAfter` | Same shape; body inserted without escaping |
| `commandSections` | Map from space-separated path (`"my-cli deploy"`) to `{ extra?, rawExtra? }` |

`manPagePlugin` also accepts `command` to rename the injected subcommand (default `"man"`).

## Output layout

The generator emits standard sections including **NAME**, **SYNOPSIS** (root only), **DESCRIPTION**, **COMMANDS** (when applicable), and **COMMAND REFERENCE** with one subsection per command path (synopsis, args, options). Flag semantics match the help plugin (including boolean `--no-*` unless `noNegate` is set).
