title: "Skills"
description: Agent skill generation for AI coding assistants.

The `@crustjs/skills` package generates and installs agent skills from your CLI's command definitions. Skills provide structured documentation that AI coding assistants (Claude Code, OpenCode) use to understand your CLI.

## Install

```sh
bun add @crustjs/skills
```

## Quick Start

Add `skillPlugin()` to your CLI. It reads `name` and `description` from your root command's `meta` — you only need to supply `version`:

```ts
import { defineCommand, runMain } from "@crustjs/core";
import { skillPlugin } from "@crustjs/skills";

const app = defineCommand({
  meta: { name: "my-cli", description: "My CLI tool" },
  plugins: [
    skillPlugin({
      version: "1.0.0",
      command: true, // registers "my-cli skill" subcommand
    }),
  ],
});

runMain(app);
```

This gives you:

- **Auto-detection**: the plugin detects which supported agents are installed by checking for their global configuration directories.
- **Auto-update**: when a user runs any command, the plugin silently updates installed skills if the version has changed.
- **Interactive command**: `my-cli skill` presents a multiselect prompt for toggling agent installations on or off.

## Plugin Options

```ts
interface SkillPluginOptions {
  /** Skill version — compared against the installed crust.json */
  version: string;
  /** Installation scope. @default "global" */
  scope?: "global" | "project";
  /** Auto-install skills when not yet present. @default false */
  autoInstall?: boolean;
  /** Auto-update skills when version is outdated. @default true */
  autoUpdate?: boolean;
  /** Register interactive skill subcommand. true = "skill", string = custom name. @default false */
  command?: boolean | string;
}
```

### Agent Auto-Detection

The plugin automatically detects which agents are installed by checking for their global configuration directories — no manual configuration needed. Only detected agents are managed by the middleware and shown in the interactive command.

| Agent | Detection Path |
| --- | --- |
| `claude-code` | `~/.claude/` |
| `opencode` | `~/.config/opencode/` |

Detection always checks global paths regardless of the installation scope.

### Auto-Install vs Auto-Update

| `autoInstall` | `autoUpdate` | Behavior |
| --- | --- | --- |
| `false` (default) | `true` (default) | Only update already-installed skills; user uses `my-cli skill` for first install |
| `true` | `true` | Fully automatic — install and update on every invocation |
| `true` | `false` | Auto-install once, never auto-update |
| `false` | `false` | Plugin does nothing automatically (command-only mode) |

### Supported Agents

| Agent | Global Path | Project Path |
| --- | --- | --- |
| `claude-code` | `~/.claude/skills/<name>/` | `.claude/skills/<name>/` |
| `opencode` | `~/.config/opencode/skills/<name>/` | `.opencode/skills/<name>/` |

## Interactive Command

When `command: true` is set, the plugin registers a `skill` subcommand that presents a single multiselect prompt.

Detected agents are listed with their current installation status. Already-installed agents are pre-selected. The user toggles agents on or off and the system reconciles the desired state:

- **Newly selected** agents are installed
- **Deselected** agents are uninstalled
- **Already-correct** agents are skipped

```sh
my-cli skill
```

## Conflict Detection

Each generated skill directory contains a `crust.json` file that serves as an ownership marker. When `generateSkill()` encounters a target directory that already exists but has no `crust.json`, it assumes the skill was not created by Crust (e.g. manually created or installed by another tool) and throws a `SkillConflictError` to prevent silent overwrites.

### Behavior by context

| Context | Conflict behavior |
| --- | --- |
| **Auto-update middleware** | Logs a warning and skips — the CLI continues running normally |
| **Interactive `skill` command** | Prompts the user to confirm overwriting the conflicting skill |
| **`generateSkill()` API** | Throws `SkillConflictError`. Pass `force: true` to overwrite |

### Handling conflicts programmatically

```ts
import { generateSkill, SkillConflictError } from "@crustjs/skills";

try {
  await generateSkill({ command, meta, agents });
} catch (err) {
  if (err instanceof SkillConflictError) {
    console.error(`Conflict at: ${err.details.outputDir}`);
    // Retry with force to overwrite
    await generateSkill({ command, meta, agents, force: true });
  }
}
```

## Low-Level Primitives

For custom workflows, the package also exports lower-level functions:

```ts
import {
  detectInstalledAgents,
  generateSkill,
  uninstallSkill,
  skillStatus,
} from "@crustjs/skills";

// Detect which agents are installed
const agents = await detectInstalledAgents();
// ["claude-code", "opencode"] — only agents with config dirs present

// Install skills
const result = await generateSkill({
  command: rootCommand,
  meta: { name: "my-cli", description: "My CLI", version: "1.0.0" },
  agents,
  scope: "global",
});

// Check status
const status = await skillStatus({
  name: "my-cli",
  agents,
});

// Uninstall
const removed = await uninstallSkill({
  name: "my-cli",
  agents,
});
```
